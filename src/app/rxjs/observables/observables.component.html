RXJS Observables

Observables are data types that allow data streams to pass through them and use suscriptions to retrieve data from the stream.

Why use observables instead of promises?

  • Observables are lazy, while promises are eager.

  • Observables can be cancelled, while promises can't.

  • Observables can be sync and async, while Promises are always resolved asynchonously

  • Observables can be paused and resumed, while promises can't.

  • Observables can be composed, while promises can't.

  • Observables can be multicasted, while promises can't. Which means they can be observed be multiple subscribers while promises can't

  • Observables can be emit multiple values, while promises can only emit one value

  • Observables have multiple operators to transform data, whilst promises can only use then, catch and finally

  • Observables can be retried, while promises can't
