RXJS Subjects

Subjects are a special type of Observable that allows values to be multicasted to many Observers. While plain Observables are unicast (each subscribed Observer owns an independent execution of the Observable), Subjects are multicast.

Subject example

const subject = new Subject();
{{ subject$ | async}}

Why use subjects instead of observables?

  • Multicasting: Observables are unicast by default. This means each subscribed observer owns an independent execution of the Observable. Subjects, on the other hand, are multicast. This means they can have multiple observers and when a value is emitted, it is passed to each observer.

  • Value emitting: Observables are passive; they emit values when they are subscribed to and the code inside the Observable function runs. Subjects, however, are active; you can manually call 'next(value)'' to emit a value.

  • Subscription: With an Observable, you can't manually add or remove observers. With a Subject, you can manually manage observers through the 'add(observer)' and 'remove(observer)' methods.

  • Replay of values: Regular Observables do not replay old values to new subscribers. Subjects, however, have variations like ReplaySubject and BehaviorSubject that can replay old values to new subscribers.

In summary, you would use an Observable when you want to create a stream of values, and a Subject when you want to multicast values to multiple observers or manually control when a value is emitted.

Behavior Subject

Is a type of subject which always has an initial value and when it gets a new subscriptor it receives the most recent value.
